# Vue 3 单页面应用工程构建原则

基于你的项目（Vue 3 + Vite + Vue Router + Axios），以下是应该遵循的核心原则：

---

## 1. 📁 目录结构原则

### ✅ 按功能模块组织，不是按文件类型

```
src/
├── api/              # API 层
│   ├── modules/      # 按业务模块拆分
│   │   ├── auth.js
│   │   ├── device.js
│   │   └── log.js
│   └── index.js      # 统一导出
│
├── stores/           # 状态管理（Pinia）
│   ├── user.js
│   ├── device.js
│   └── index.js
│
├── views/            # 页面组件
│   ├── device/       # 设备相关页面
│   │   ├── List.vue
│   │   ├── Detail.vue
│   │   └── components/  # 页面私有组件
│   └── log/
│
├── components/       # 全局公共组件
│   ├── common/       # 通用组件（Button, Input）
│   └── business/     # 业务组件（DeviceCard）
│
├── composables/      # 组合式函数（Vue 3 推荐）
│   ├── useAuth.js
│   └── useWebSocket.js
│
├── utils/            # 工具函数
│   ├── request.js    # axios 封装
│   ├── format.js     # 格式化工具
│   └── validate.js   # 验证工具
│
└── router/           # 路由配置
    └── index.js
```

**原则**: 相关的东西放在一起，降低认知负担

---

## 2. 🔌 API 层原则

### ✅ 统一管理，按模块拆分

```javascript
// ❌ 错误：在组件中直接调用 axios
axios.post('/api/device/list')

// ✅ 正确：通过 API 层调用
import { getDeviceList } from '@/api/modules/device'
getDeviceList()
```

**原则**:
- API 调用集中管理
- 便于修改和维护
- 便于 Mock 测试

---

## 3. 🗄️ 状态管理原则

### ✅ 全局状态用 Pinia，局部状态用 ref/reactive

```javascript
// ✅ 全局状态（跨组件共享）
const userStore = useUserStore()

// ✅ 局部状态（组件内部）
const loading = ref(false)
```

**什么放 Store**:
- ✅ 用户信息
- ✅ 权限数据
- ✅ 全局配置
- ✅ 跨页面共享的数据

**什么不放 Store**:
- ❌ 表单数据
- ❌ 列表数据（除非需要缓存）
- ❌ 临时状态

---

## 4. 🎨 组件设计原则

### ✅ 单一职责 + 组合优于继承

```vue
<!-- ❌ 错误：一个组件做太多事 -->
<DeviceManagement>
  <!-- 搜索、列表、分页、编辑、删除都在一个组件 -->
</DeviceManagement>

<!-- ✅ 正确：拆分职责 -->
<DeviceList>
  <DeviceSearch />
  <DeviceTable />
  <DevicePagination />
</DeviceList>
```

**原则**:
- 一个组件只做一件事
- 可复用的提取为公共组件
- 复杂逻辑提取为 composables

---

## 5. 🔄 数据流原则

### ✅ 单向数据流

```
用户操作 → Action → API → Store → View
```

```javascript
// ✅ 正确的数据流
const handleDelete = async (id) => {
  await deleteDevice(id)        // 1. 调用 API
  deviceStore.removeDevice(id)  // 2. 更新 Store
  // 3. View 自动响应更新
}

// ❌ 错误：直接修改 props
props.device.name = 'new name'
```

---

## 6. 🛠️ 工具函数原则

### ✅ 纯函数 + 单一职责

```javascript
// ✅ 正确：纯函数
export function formatDate(date) {
  return dayjs(date).format('YYYY-MM-DD')
}

// ❌ 错误：有副作用
export function formatDate(date) {
  console.log(date)  // 副作用
  localStorage.setItem('lastDate', date)  // 副作用
  return dayjs(date).format('YYYY-MM-DD')
}
```

**原则**:
- 输入相同，输出相同
- 无副作用
- 易于测试

---

## 7. 🎯 Composables 原则（Vue 3 核心）

### ✅ 提取可复用逻辑

```javascript
// composables/useDeviceList.js
export function useDeviceList() {
  const devices = ref([])
  const loading = ref(false)
  
  const fetchDevices = async () => {
    loading.value = true
    devices.value = await getDeviceList()
    loading.value = false
  }
  
  return { devices, loading, fetchDevices }
}

// 在组件中使用
const { devices, loading, fetchDevices } = useDeviceList()
```

**什么提取为 Composable**:
- ✅ 可复用的业务逻辑
- ✅ 有状态的逻辑
- ✅ 需要生命周期的逻辑

## 8. 🔐 环境配置原则

### ✅ 使用环境变量，不要硬编码

```javascript
// ❌ 错误：硬编码
const API_URL = 'http://172.27.135.60:8080'

// ✅ 正确：环境变量
const API_URL = import.meta.env.VITE_API_BASE_URL
```

**文件结构**:
```
.env.development    # 开发环境
.env.production     # 生产环境
.env.local          # 本地覆盖（不提交 git）
```

---

## 9. 🚦 路由设计原则

### ✅ 扁平化 + 懒加载

```javascript
// ✅ 正确：懒加载
{
  path: '/device',
  component: () => import('@/views/device/List.vue')
}

// ❌ 错误：同步加载
import DeviceList from '@/views/device/List.vue'
{
  path: '/device',
  component: DeviceList
}
```

**原则**:
- 所有路由懒加载
- 路由层级不超过 3 层
- 使用 meta 存储路由元信息

## 10. 📦 依赖管理原则

### ✅ 最小化依赖

**当前项目依赖**:
```json
{
  "vue": "^3.5.25",           // ✅ 必需
  "vue-router": "^4.6.3",     // ✅ 必需
  "axios": "^1.13.2"          // ✅ 必需
}
```

**建议添加**:
```json
{
  "pinia": "^2.x",            // 状态管理
  "element-plus": "^2.x"      // UI 库（可选）
}
```

**原则**:
- 能不装就不装
- 优先使用原生 API
- 大型库按需引入

## 11. 🎨 样式管理原则

### ✅ CSS 变量 + Scoped

```css
/* styles/variables.css */
:root {
  --color-primary: #3498db;
  --color-success: #2ecc71;
  --spacing-base: 8px;
}

/* 组件中使用 */
<style scoped>
.button {
  color: var(--color-primary);
  padding: var(--spacing-base);
}
</style>
```

**原则**:
- 全局变量统一管理
- 组件样式使用 scoped
- 公共样式提取到 common.css

## 12. 🐛 错误处理原则

### ✅ 统一处理 + 分层处理

```javascript
// 1. axios 拦截器统一处理
service.interceptors.response.use(
  response => response,
  error => {
    // 统一错误提示
    message.error(error.message)
    return Promise.reject(error)
  }
)

// 2. 组件中处理特殊情况
try {
  await deleteDevice(id)
} catch (error) {
  // 特殊处理
  if (error.code === 'DEVICE_IN_USE') {
    message.warning('设备使用中，无法删除')
  }
}
```

## 13. 📝 代码规范原则

### ✅ 统一风格

```javascript
// ✅ 使用 ESLint + Prettier
npm run lint
npm run format

// ✅ 命名规范
- 组件: PascalCase (DeviceList.vue)

```

## 14. 🧪 可测试性原则

### ✅ 便于测试的代码

```javascript
// ✅ 正确：纯函数，易测试
export function calculateTotal(items) {
  return items.reduce((sum, item) => sum + item.price, 0)
}

// ❌ 错误：依赖外部状态，难测试
export function calculateTotal() {
  return store.items.reduce((sum, item) => sum + item.price, 0)
}
```

## 🎯 你的项目应该立即遵循的原则

### 优先级 1（立即执行）
1. ✅ **修复 axios 响应拦截器**（返回 response 而不是 response.data）
2. ✅ **添加环境变量配置**（.env 文件）
3. ✅ **引入 Pinia**（替代 localStorage 管理状态）

### 优先级 2（本周完成）
4. ✅ **重构 API 层**（按模块拆分）
5. ✅ **统一错误处理**
6. ✅ **提取公共逻辑为 composables**

### 优先级 3（长期优化）
7. ✅ **添加 TypeScript**
8. ✅ **引入 UI 库**（Element Plus）
9. ✅ **添加单元测试**

---

## 📚 核心思想总结

1. **关注点分离**: API、状态、视图、逻辑分离
2. **单一职责**: 一个模块只做一件事
3. **可复用性**: 提取公共逻辑和组件
4. **可维护性**: 统一规范，便于理解
5. **可测试性**: 纯函数，低耦合
6. **性能优化**: 懒加载，按需引入

**记住**: 好的工程不是一次性完成的，而是持续重构优化的结果。
