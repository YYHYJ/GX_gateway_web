# GX Gateway Web 工程问题分析报告

## 🔴 严重问题

### 1. **响应拦截器返回错误** ⚠️ 高优先级
**位置**: `src/utils/request.js:38`

```javascript
// ❌ 错误：返回 response.data
return response.data

// ✅ 应该返回完整的 response
return response
```

**问题**: 
- 组件中使用 `response.data.code` 会失败，因为已经返回了 `data`
- 导致需要写 `response.code` 而不是标准的 `response.data.code`
- 不一致的 API 调用方式

**影响范围**: 所有使用 axios 的地方

---

### 2. **缺少状态管理** ⚠️ 高优先级
**位置**: `src/store/` 目录为空

**问题**:
- 用户信息存储在 `localStorage`，没有响应式
- 多个组件重复获取用户信息
- 无法统一管理全局状态（用户、权限、配置等）

**建议**: 引入 Pinia（Vue 3 官方推荐）

---

### 3. **环境变量配置缺失** ⚠️ 中优先级
**位置**: `vite.config.js` 和根目录

**问题**:
- 后端地址硬编码在 `vite.config.js`
- 缺少 `.env.development` 和 `.env.production`
- 无法区分开发/生产环境

**建议结构**:
```
.env.development
.env.production
.env.local (git ignore)
```

## 🟡 中等问题

### 4. **路由守卫逻辑混乱**
**位置**: `src/router/index.js:120-145`

**问题**:
- 登录状态只检查 `localStorage.getItem('isLoggedIn')`
- 没有验证 token 有效性
- 路由守卫中有异步操作但没有正确处理

**建议**:
```javascript
router.beforeEach(async (to, from, next) => {
  if (to.meta.requiresAuth) {
    const token = localStorage.getItem('token')
    if (!token) {
      return next('/')
    }
    // 可选：验证 token 有效性
    try {
      // await validateToken()
      next()
    } catch {
      localStorage.clear()
      next('/')
    }
  } else {
    next()
  }
})
```

**src/stores/user.js**
```javascript
import { defineStore } from 'pinia'

export const useUserStore = defineStore('user', {
  state: () => ({
    token: localStorage.getItem('token') || '',
    userInfo: JSON.parse(localStorage.getItem('userInfo') || '{}'),
  }),
  
  getters: {
    isLoggedIn: (state) => !!state.token,
  },
  
  actions: {
    setToken(token) {
      this.token = token
      localStorage.setItem('token', token)
    },
    
    logout() {
      this.token = ''
      this.userInfo = {}
      localStorage.clear()
    }
  }
})
```
